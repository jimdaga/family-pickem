<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Untrusted uploaded bytes -> Pillow decoder | A file can spoof its name, MIME header, image signature, dimensions, or trailing content. |
| Decoded source -> canonical output | Metadata and source bytes must not survive into the asset eligible for later storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-06-01 | Tampering | uploaded image bytes | high | mitigate | Restrict Pillow decoder formats, verify then reopen/load, and persist only a fresh WebP later. |
| T-06-02 | Denial of service | decoder/crop transform | high | mitigate | Enforce 5 MiB before decoder work; after `verify()` restricted-reopen and reject >16 MP dimensions before `load()`/any transform; promote decompression warnings and validate crop bounds before transform. |
| T-06-03 | Information disclosure | image metadata | medium | mitigate | Encode from a fresh image buffer with no copied metadata and test sentinel removal. |
| T-06-SC | Supply chain | image dependencies | medium | mitigate | Use already-pinned Pillow only; do not install packages. |
</threat_model>
