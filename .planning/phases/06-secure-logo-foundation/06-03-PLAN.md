<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| browser multipart POST -> pre-CSRF route middleware -> tenant settings view | File fields and every form value are untrusted; the handler must be installed before CSRF can parse multipart data, while only route-derived tenant context authorizes mutation. |
| settings view -> canonical processor -> storage | The view must pass only safe output to the field and cannot persist source bytes. |
| template -> browser | UI may guide selection but cannot assert that a browser-claimed type or local preview is safe. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-06-08 | Elevation of privilege | logo settings POST | high | mitigate | Reuse `family_member_required(admin)`, CSRF middleware, route-derived tenant context, and locked rows. |
| T-06-09 | Tampering | form/storage authority | high | mitigate | Accept one source file only; family, key, URL, filename, and output type remain server-owned. |
| T-06-10 | Tampering | invalid source replacement | high | mitigate | Process before assignment and retain the prior reference on a validation error. |
| T-06-11 | Information disclosure | settings/audit/template output | medium | mitigate | Render canonical/default imagery only and record boolean state, never storage internals or source metadata. |
| T-06-12 | Denial of service | multipart source | high | mitigate | nginx has a 6m request-body ceiling to accommodate multipart framing, while the application retains the exact 5 MiB source-file ceiling; route-aware middleware immediately before CSRF installs a streaming handler which records `file_too_large` then raises `StopUpload` over 5 MiB before CSRF/Django spools a temporary file, Pillow processes, or storage persists it. The view consumes only that marker to render a safe bound-field error; Plan 01 limits remain defense in depth. |
| T-06-13 | Tampering / repudiation | logo storage-to-DB persistence | high | mitigate | After the storage write caused by `logo.save(..., save=False)`, persist the locked row with `update_fields` including `logo` before boolean-only audit metadata. Compensate a DB/audit failure by best-effort deletion of only the newly generated key; retain the prior DB reference and never audit an unpersisted change. |
</threat_model>
