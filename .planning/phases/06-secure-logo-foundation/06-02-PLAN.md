<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| ORM file name -> storage backend | Model values can become cloud object paths unless the key is generated exclusively by application code. |
| legacy URL database value -> migration | Historic arbitrary URLs are untrusted network references, not upload data. |
| application storage -> S3 | The app’s configured bucket is private; the model must not use static storage or grant browser write authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-06-04 | Tampering | object key generation | high | mitigate | Ignore original filename and derive a UUID WebP name from the saved family id. |
| T-06-05 | SSRF/tampering | legacy URL migration | high | mitigate | Remove URL values without fetching, validating, or copying their targets. |
| T-06-06 | Information disclosure | storage configuration | high | mitigate | Use dedicated private signed storage under the existing private bucket; no public ACL/policy changes. |
| T-06-07 | Denial of service | tests | medium | mitigate | Use local/fake storage and assert no live cloud calls. |
</threat_model>
