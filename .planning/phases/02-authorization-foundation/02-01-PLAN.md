# Phase 02 Plan 01: Core Tenant Authorization Helpers

## Goal

Create the canonical server-side helper layer for resolving family/pool context, active memberships, and role checks. This plan should not migrate product pages yet.

## Scope

- Add `pickem/pickem_api/authz.py`.
- Define role ordering for `member`, `admin`, and `owner`.
- Define predictable denial types for auth-required, not-found/non-member, and permission-denied cases.
- Resolve active `FamilyMembership` by user and family.
- Resolve pool context and enforce `pool.family_id == family.id` when family context is supplied.
- Add optional legacy default pool resolution only when explicitly requested by the caller.
- Add unit tests proving helper behavior and cross-family denial.

## Out of Scope

- Broad page/API tenant filtering.
- Onboarding, invitations, family switcher, and admin UI.
- Changing `is_commissioner()` or `commissioner_required()`.
- Database migrations unless a missing index is discovered and justified.

## Files Likely To Change

- `pickem/pickem_api/authz.py` (new)
- `pickem/pickem_api/tests.py` or `pickem/pickem_api/tests/test_authz.py`
- Optional: `pickem/pickem_api/__init__.py` only if exports are needed

## Tasks

1. Add failing tests for active member, admin, owner, inactive member, outsider, anonymous user, superuser without membership, and commissioner without membership.
2. Add role hierarchy helpers and denial exceptions/results in `pickem_api.authz`.
3. Implement family resolution by object, id, or slug as needed by tests.
4. Implement pool resolution by object/id/slug with family consistency checks.
5. Implement explicit legacy default pool fallback helper, with tests showing it is opt-in.
6. Run focused `pickem_api` tests and commit the plan implementation atomically.

## Security Considerations

- Never trust client-supplied family/pool identifiers without membership resolution.
- Treat inactive memberships as no membership.
- Do not allow `is_superuser`, `is_staff`, or `UserProfile.is_commissioner` to bypass tenant checks.
- Avoid error strings that reveal whether an inaccessible family/pool exists.
- Keep default legacy fallback explicit so future route migrations cannot accidentally depend on global data.

## Tests To Add/Update

- `require_family_membership` returns active membership for member/admin/owner.
- `require_family_membership(..., minimum_role='admin')` denies member.
- inactive membership denied as not found.
- outsider denied as not found.
- anonymous denied as authentication-required.
- superuser without membership denied.
- commissioner without membership denied.
- `require_pool_membership` denies family/pool mismatch.
- family A member cannot resolve family B pool.

## Manual QA Checklist

- No manual QA required for this helper-only plan.
- Confirm no routes were broadly migrated.

## Rollback Considerations

Rollback is low risk because this plan only adds helper code/tests. Revert the commit if helper API shape proves wrong before downstream phases depend on it.

## Definition Of Done

- Central helper module exists and is covered by negative tests.
- Cross-family and wrong-role denials are tested.
- Focused `pickem_api` tests pass.
