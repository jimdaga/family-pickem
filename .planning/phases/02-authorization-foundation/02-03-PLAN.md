# Phase 02 Plan 03: Verification And Handoff

## Goal

Verify the authorization foundation end to end and leave clear handoff notes for onboarding and route migration phases.

## Scope

- Run focused and full Django verification.
- Run migration check to ensure Phase 2 did not accidentally require schema changes.
- Update Phase 2 summary with helper API, tests, and remaining route migration risks.
- Update planning state/roadmap to reflect completed Phase 2 implementation.

## Out of Scope

- Fixing unrelated existing warnings.
- Migrating user-facing family context.
- Production deployment changes.

## Files Likely To Change

- `.planning/phases/02-authorization-foundation/02-03-SUMMARY.md`
- `.planning/STATE.md`
- `.planning/ROADMAP.md`
- Possibly `TEST_PLAN.md` if execution reveals important new test commands or risks

## Tasks

1. Run `cd pickem && ../venv/bin/python manage.py check --settings=pickem.test_settings`.
2. Run `cd pickem && ../venv/bin/python manage.py makemigrations --check --dry-run --settings=pickem.test_settings`.
3. Run focused `pickem_api` tests.
4. Run focused `pickem_homepage` tests.
5. Run full Django test suite.
6. Write Phase 2 summary with commands, results, helper APIs, and open risks.
7. Commit verification/docs atomically.

## Security Considerations

- Ensure every denied-path test is still present and passing before marking Phase 2 complete.
- Record any route still using global data so Phase 4/5 cannot miss it.
- Confirm no helper permits superuser/commissioner bypass.

## Tests To Add/Update

No new product tests should be needed in this plan unless verification finds a missing negative case. If a missing case appears, add it before summary.

## Manual QA Checklist

- Optional: exercise the proof endpoint/page locally if it exists.
- Confirm homepage still renders after helper additions.
- Confirm no unexpected migrations are generated.

## Rollback Considerations

This plan is primarily verification/docs. If full-suite regression appears, fix or revert the specific implementation commit from Plan 01/02 before completing this plan.

## Definition Of Done

- Django check passes or only existing unrelated warnings remain.
- No pending migrations.
- Focused and full tests pass.
- Phase 2 summary and state are updated.
